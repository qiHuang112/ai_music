# AM-20260717-001 产品需求 R2：批量完整音频搜索聚合

Status: User approved through DISC-0008 and DISC-0009

## 用户目标

用户搜索歌手或歌曲时，应立即获得本机已有完整音乐，并在不降低完整音频门槛的前提下看到一批可用在线结果。即使歌曲海在手机网络上不可达，缓存和其他已准入来源仍应独立工作；系统可以低频评估更多来源，但不能以网盘、试听、防护绕过或密集重试换取结果数量。

## 真实核心路径

1. 提交搜索后，先同步匹配索引有效且已经完整落盘的本地缓存，立即展示并可本地播放。
2. 在线来源独立搜索和严格校验；首屏以批量结果为目标，最多串行预取 3 个聚合页，达到 8 首或来源耗尽即停止。
3. 本地与在线结果按标准化歌名和歌手去重，稳定追加到当前搜索歌曲库与 media session 队列。
4. 首屏有界预取结束后，仅在用户接近列表底部时继续按需分页。
5. 单来源失败或熔断时保留本地及其他来源结果，不污染缓存。

## 必须范围

- 本地完整缓存即时聚合，支持歌手、歌曲名和歌手加歌曲名匹配；本地命中不产生网络请求。
- 歌曲海与酷我继续作为当前在线聚合来源，保持独立分页、错误和两分钟熔断。
- 首屏最多 3 个聚合页或 8 首的有界批量策略；同一来源分页串行，已有行不重排。
- 所有在线可见结果保持 `directAudio`、`canCacheAudio`、`clientReady`、音频 HEAD 和总长一致 Range 206 的完整准入。
- 额外来源只做低频严格准入：每来源最多一个探测任务在途；只有真机可达并通过同等完整音频门槛后，才能另行加入生产聚合。

## 非目标

- 不要求本次必须接入 BuguYY、FLAC、22a5 或固定数量的新来源；当前未通过真机准入的来源继续 fail closed。
- 不接入 PREVIEW、网盘、HTML、防护页、浏览器专用地址，不绕过验证码、证书错误或第三方安全验证。
- 不无限预取页面；首屏有界预取后仍由用户触发后续分页。
- 不新增来源筛选页、来源设置或新的播放模式。
- 不回滚或重开 AM-20260711-005。

## 验收标准

1. 搜索提交后，所有匹配的有效本地完整缓存立即出现并可直接播放，本地命中不发网络请求；后到在线同曲不得覆盖或重复本地行。
2. 当有效候选充足时，首屏无需反复下拉即可批量呈现结果：系统最多串行请求 3 个聚合页，并在达到 8 首、来源耗尽或页数上限任一条件满足时停止；单条中间结果不得被误报为首屏聚合完成。
3. 歌曲海、酷我及未来获准来源均独立分页和熔断；超时、403、429、防护或安全验证触发两分钟单源熔断，其他来源与本地结果继续可搜索、分页和播放。
4. 在线可见结果全部通过可信匹配、`directAudio/canCacheAudio/clientReady`、音频 HEAD 和总长一致 Range 206；额外来源未通过小米真机低频准入前不得进入列表，PREVIEW、网盘、HTML、防护页和低置信结果不可见。
5. 动态结果稳定去重并进入搜索歌曲库与 media session 队列，不改变当前歌曲、位置和播放模式；未播放在线候选不创建 progressive session、part 文件或正式缓存，失败不污染缓存，并以小米 10 Pro 的周杰伦、王蓉主路径及测试、APK SHA 证据验收。
