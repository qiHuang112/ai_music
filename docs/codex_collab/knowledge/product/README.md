这里沉淀 AI Music 的产品反馈、用户场景、交互取舍、验收口径、优先级判断和后续测试/UI lane 招募记录。

## Lane 身份

- Lane: `product`
- 线程标题：`AIMusic-产品负责人`
- Thread ID: `019eea5b-9b46-7f92-a35c-7d080ea1e986`

## 工作边界

- 产品 lane 负责说明“为什么做、做到什么程度、什么算好用”。
- 架构师 lane 负责把产品意图拆成任务、协调开发 lane、review 和验收流转。
- 开发 lane 不清楚体验取舍时，先回到产品 lane 确认，不自行扩大范围。
- 测试和 UI lane 后续加入时，先读取本目录，再读取相关 request 和 review 记录。

## 重点沉淀方向

- 用户原始反馈和复盘后的产品判断。
- 功能优先级、延期项和明确不做的范围。
- 关键交互决策，例如入口、默认状态、确认弹窗、批量操作、排序规则。
- 验收标准和测试关注点。
- 后续测试/UI lane 的招募、职责和协作方式。
